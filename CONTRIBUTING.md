# Contributing

Thanks for helping improve the React Training Lab. This is a learning sandbox, so keep changes small and focused.

## Quick workflow
1. Create or pick an issue.
2. Create a branch for the ticket.
3. Fix one thing at a time.
4. Open a PR and ask for review.
5. If a bug is fixed, update `src/data/bugs.js` to `status: 'Fixed'`.

## Branch naming
- bug/BUG-101-burst-counter
- feat/FEAT-201-quick-notes
- chore/docs-readme

## Commit messages
- fix: BUG-101 burst counter
- feat: FEAT-201 quick notes
- docs: update README

## Labels
We keep a recommended list in `.github/labels.yml`.
Use `bug`, `feature`, and `beginner` for most tickets.

## Pull requests
Keep PRs small and focused. Add screenshots when UI changes.
